# Стратегия развития ATP Platform: двухтрековый запуск

**Дата:** 2026-03-28
**Статус:** Утверждён
**Срок:** ~10 недель (3 фазы)
**Команда:** 2–3 человека

## Контекст

ATP (Agent Test Platform) — фреймворк-агностичная платформа для тестирования и оценки AI-агентов. Проект завершил фазы 1–5 (декомпозиция, экосистема плагинов, операционная часть). Он включает 5 workspace-пакетов (atp-core, atp-adapters, atp-dashboard, game-environments, atp-games), 9+ адаптеров, 10+ типов оценщиков (evaluators), ~250 тестов и обширную документацию.

### Цели

- **Публичный релиз и привлечение пользователей** — публикация в PyPI, полировка DX, привлечение аудитории
- **Исследовательский инструмент** — теоретико-игровая оценка LLM-агентов, воспроизводимые эксперименты

### Целевая аудитория

В равной мере — AI-исследователи и AI-инженеры в продуктовых командах.

### Конкурентное преимущество

Сочетание протокол-агностичного тестирования (единый протокол для любого агентного фреймворка) и теоретико-игровой оценки уникально. Ни один существующий инструмент (DeepEval, Promptfoo, Inspect AI) не предлагает обоих возможностей.

### Ограничения

- Стабильность протокола требует аудита перед публичным коммитом
- Команда из 2–3 человек — необходимо эффективно параллелить работу
- 29 существующих ошибок pyrefly, которые нужно исправить

---

## Стратегия: два трека в 3 фазы

Два параллельных рабочих трека, сходящихся в единый запуск.

| Фаза | Длительность | Трек 1: Платформа | Трек 2: Исследования |
|------|-------------|-------------------|----------------------|
| **Фаза 0: Стабилизация** | 3 недели | Аудит протокола, исправление pyrefly, аудит API-поверхности | Аудит теории игр, граничные случаи, выявление пробелов |
| **Фаза 1: Публикация** | 4 недели | Публикация в PyPI, полировка CLI/DX, quickstart, Docker | Новые игры, многораундовые турниры, LLM-бенчмарки |
| **Фаза 2: Запуск** | 3 недели | Шаблоны CI/CD, Docker-образы, документация, лицензирование | Визуализации, воспроизводимые эксперименты, блог-пост |

**Точка синхронизации:** конец Фазы 1 — интеграционный тест с опубликованными в PyPI пакетами + турнир через CLI + дашборд.

## Фаза 0: Стабилизация (3 недели)

### Трек 1: Аудит и стабилизация протокола

- **Аудит ATP-протокола:** ревью ATPRequest/ATPResponse/ATPEvent на полноту и консистентность. Проверить: все ли поля необходимы, нет ли дублирования, корректны ли значения по умолчанию.
- **Семантическое версионирование:** заморозить Protocol v1.0. Задокументировать, что считается ломающим изменением. Добавить поле версии протокола, если отсутствует.
- **Очистка pyrefly:** исправить 29 существующих ошибок типизации, настроить CI для блокировки новых.
- **Аудит API-поверхности:** ревью публичного API каждого пакета (atp-core, atp-adapters, atp-dashboard). Убедиться, что `__all__`-экспорты корректны, внутренние модули помечены `_private`.
- **Зачистка устаревшего:** найти и удалить мёртвый код, неиспользуемые фичи, заглушки TODO.

### Трек 2: Аудит теории игр

- **Покрытие игр:** аудит 5 существующих игр и 19 стратегий — проверка матриц выигрышей, корректности равновесий Нэша.
- **Граничные случаи:** ревью покрытия property-based тестов hypothesis для граничных условий (ноль раундов, идентичные стратегии, экстремальные выигрыши).
- **Стабильность LLM-агента:** проверка надёжности агента GPT-4o-mini в «Дилемме заключённого» — логика повторов, обработка таймаутов, отслеживание стоимости.
- **Выявление пробелов:** каких классических игр не хватает для убедительной исследовательской истории.

### Результаты фазы

- Документ «Спецификация ATP Protocol v1.0» в `docs/`
- Документ «Отчёт о покрытии теории игр» в `docs/`

## Фаза 1: Публикация (4 недели)

### Трек 1: PyPI и Developer Experience

**Порядок публикации:**
1. `game-environments` (standalone, без зависимости от ATP)
2. `atp-games` (зависит от game-environments + pydantic)
3. `atp-core`, `atp-adapters`, `atp-dashboard` (workspace-члены)
4. `atp-platform` (мета-пакет — обновить зависимости на PyPI-версии подпакетов)

**Полировка CLI/DX:**
- `atp init` — генерирует рабочий проект одной командой (тестовый набор + конфиг + пример агента)
- `atp quickstart` — интерактивный мастер: выбор адаптера, генерация тестового набора, первый запуск
- Улучшение сообщений об ошибках — понятные подсказки вместо трейсбеков
- Единообразные флаги `--verbose` / `--quiet` во всех командах

**Руководство для быстрого старта:** «От нуля до первого теста за 5 минут» — установка через pip, запуск демо-агента, просмотр результатов.

**Docker:** официальный `Dockerfile` + `docker-compose.yml` с дашбордом + демо-агентом для мгновенного запуска.

### Трек 2: Расширение теории игр

**Новые игры (2–3):**
- **Игра об общественном благе (Public Goods Game)** — тестирует групповую кооперацию (>2 агентов)
- **Битва полов (Battle of the Sexes)** — тестирует координацию при конфликте предпочтений
- **Охота на оленя (Stag Hunt)** — тестирует доверие vs безопасность (дополняет «Дилемму заключённого»)

**Многораундовые турниры:**
- Round-robin между N агентами × M стратегиями
- Elo-рейтинг агентов по результатам турниров
- Персистентность: история турниров сохраняется в БД дашборда

**LLM-агентные бенчмарки:**
- Стандартный набор: 3–5 моделей (GPT-4o-mini, Claude Haiku, Gemini Flash, Llama) × 5 игр × 100 раундов
- Метрики: уровень кооперации, скорость адаптации, консистентность, стоимость за игру
- Воспроизводимость: рандомизация на основе seed, фиксированные промпты

### Точка синхронизации

Интеграционный тест в конце Фазы 1: установить все пакеты из PyPI, запустить турнир через CLI, проверить отображение результатов в дашборде.

## Фаза 2: Запуск (3 недели)

### Трек 1: CI/CD и готовность к продакшену

**Шаблоны GitHub Actions:**
- `atp-test.yml` — запуск тестового набора в CI, проверка регрессии по бейслайну
- `atp-game-eval.yml` — теоретико-игровая оценка как шаг CI
- Готовые к копированию примеры для любого репозитория

**Docker-образы:**
- `ghcr.io/*/atp-runner` — минимальный образ для CI
- `ghcr.io/*/atp-dashboard` — дашборд с предзагруженными демо-данными
- Мультиархитектурность (amd64 + arm64)

**Документация к запуску:**
- Переписать README: ясное ценностное предложение — «ATP = фреймворк-агностичное тестирование агентов + теоретико-игровая оценка»
- Страница сравнения: ATP vs DeepEval vs Promptfoo vs Inspect AI — честная таблица с акцентом на уникальности
- Руководство для контрибьюторов: как добавить адаптер, оценщик, игру

**Лицензирование:** убедиться, что файл LICENSE существует, выбрать лицензию (MIT/Apache 2.0), проверить совместимость зависимостей.

### Трек 2: Исследовательский контент и визуализации

**Визуализации:**
- Интерактивные графики турниров (тепловая карта кооперации, прогресс Elo, распределение стратегий)
- Экспорт в PNG/SVG для публикаций
- Интеграция с дашбордом: новая страница «Аналитика игр»

**Воспроизводимые эксперименты:**
- `examples/experiments/` — полный пайплайн: конфиг, запуск, результаты, визуализация
- `atp experiment run examples/experiments/llm_prisoners_dilemma.yaml` — одна команда
- Результаты в машиночитаемом формате (JSON + CSV)

**Блог-пост / статья:**
- «Насколько кооперативен ваш AI? Теоретико-игровая оценка LLM-агентов»
- Содержание: мотивация, методология, результаты бенчмарков, как воспроизвести
- Целевые площадки: dev.to, Medium, arxiv (если достаточно глубоко)

## Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Ломающие изменения протокола после публикации | Средняя | Высокое | Аудит в Фазе 0 + версионирование протокола. Исправить проблемы ДО публикации в PyPI |
| Стоимость LLM API при бенчмарках | Высокая | Среднее | Использовать дешёвые модели (Haiku, Flash, GPT-4o-mini). Бюджетные лимиты через atp cost. Кэширование результатов для повторных запусков |
| Размытие фокуса между треками | Средняя | Среднее | Еженедельная синхронизация. Чёткое разделение ответственности. Точка синхронизации в Фазе 1 обязательна |
| Ад зависимостей PyPI | Низкая | Высокое | Тестирование установки в чистом venv на каждом этапе публикации. Минимизация транзитивных зависимостей |
| Низкое adoption после запуска | Средняя | Высокое | Блог-пост + Hacker News, Reddit r/MachineLearning. Примеры для популярных фреймворков (LangGraph, CrewAI) в quickstart |

## Метрики успеха

### На момент запуска (~10 недель):

- **Платформа:** все 5 пакетов в PyPI, `pip install atp-platform && atp quickstart` работает менее чем за 5 минут
- **Исследования:** как минимум 3 модели протестированы в 5 играх, результаты опубликованы и воспроизводимы
- **Документация:** quickstart, страница сравнения, руководство контрибьютора, 1 блог-пост
- **Качество:** 0 ошибок pyrefly, покрытие тестами >= 80%, CI зелёный

### 3 месяца после запуска:

- Загрузок с PyPI > 500/месяц
- Звёзд на GitHub > 100
- Как минимум 2 внешних контрибьютора
- Как минимум 1 внешняя публикация/блог-пост об ATP
